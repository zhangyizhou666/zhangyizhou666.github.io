Method

System design

Source layer ICU corpus

Normalized public ICU pages, portal-derived records, and eHandbook-derived records.

→
Index layer Chunks + embeddings

Passage chunks are represented through lexical terms and BGE-M3 vectors in PostgreSQL/pgvector.

→
Retrieval layer Hybrid + entity-aware RRF

Lexical, dense, Japanese-aware, and entity-alias candidates are merged with reciprocal rank fusion.

→
Generation layer Cited answer

Small or API-hosted models answer from the retrieved top evidence.

→
Evaluation layer RAGAS + human review

Outputs are evaluated with RAGAS, basic runtime checks, retrieval-side checks, and planned human review.

Corpus diagnostic

3D embedding map

The latest retrieval index contains 30,216 passage chunks. We project the real BGE-M3 embedding vectors into three PCA dimensions to inspect source types, course chunks, PDF chunks, and other corpus regions interactively.

Evaluation

Final-round evaluation structure

The current final controlled run freezes the retrieval system first, then compares answer models from the same cached Top-5 evidence. The May 1 results remain useful as pilot evidence, but the thesis tables should use the May 7 controlled setup with audited gold references.

Frozen retrievalBGE-M3 + PostgreSQL

The final run uses local PostgreSQL/pgvector and a saved retrieval cache.

Controlled matrix7 model/mode cells

Qwen3.5 and Gemma 4 variants are tested with one grounded prompt style; only thinking mode changes where required.

Layered evaluationRAGAS + human rubric

RAGAS remains diagnostic; answer acceptance, readability, citation usefulness, and over-answering risk handle user-facing quality.
