Harder Benchmark Questions

The initial 50-question benchmark was kept as a baseline, and a harder 20-question benchmark was added to expose more realistic RAG failures.

evaluationbenchmarkquestions

At a glance

  • Research question: Are the current benchmark questions strong enough to show practical differences between small answer models?
  • What changed: We kept the earlier 50-question Gemma/Qwen run and added a harder 20-question benchmark with more realistic student questions.
  • Main result: The harder benchmark is more useful for finding failure modes. The initial 50-question set produced scores that were too close to support a clear model-size claim.
  • Current limitation: The evidence still needs manual review. RAGAS scores alone are not enough for the thesis claim.

Method

Two benchmark sets were used.

  • The initial 50-question set tested broad ICU questions with Gemma 3 and Qwen3.5 answers.
  • The hard realistic 20-question set tested Qwen3.5 0.8B, 2B, and 4B on questions closer to actual student information needs.
  • Each run was interpreted with both plots and saved answer examples, so the scores could be checked against real model behavior.

Detailed benchmark pages:

  • Initial 50 Benchmark
  • Hard Realistic 20

Results

The initial 50-question run is useful as a historical baseline, but it does not separate the tested models clearly enough.

The harder 20-question run gives a better test of practical behavior. The 4B model is stronger on answer correctness, but the result does not support a simple claim that every metric improves linearly with model size.

Interpretation

The main conclusion is about benchmark quality. The first question set was too broad for the current research goal. The harder question set is a better basis for later controlled comparisons because it creates more visible failures.

This supports a cautious direction: keep retrieval fixed, improve question quality, and add human-facing review before making a strong model comparison claim.

Next step

  • Add human evaluation criteria for usefulness, citation clarity, and language consistency.
  • Keep the harder benchmark as the main public evaluation track.
  • Decide whether the next system change should target prompting, chunking, or model adaptation.
