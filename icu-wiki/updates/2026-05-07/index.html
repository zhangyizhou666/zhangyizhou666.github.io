Final Controlled Evaluation

The active final baseline uses GPT Pro gold answers and GPT Pro human-facing judge scores over one frozen retrieval setup.

final runevaluationanswerability

At a glance

  • Research question: Which local model settings produce usable student-facing answers when every model receives the same retrieved evidence?
  • What changed: The older gold/evaluation layer was replaced with GPT Pro gold answers and GPT Pro human-facing judge scores.
  • Main result: Qwen3.5 4B non-thinking is the strongest Qwen setting in this run. The result should still be read with latency, answer length, and failure tags, not one score alone.
  • Current limitation: GPT Pro references are research references created from retrieved evidence. They are not official ICU answers.

Method

This update reports the active final controlled evaluation data.

  • Run ID: final_controlled_v2_20260507T114145Z.
  • Runtime: local GPU evaluation environment with local PostgreSQL and pgvector.
  • Corpus: structural_fallback_v1, 30,216 passages.
  • Retrieval embedding: local BAAI/bge-m3.
  • Retrieval stack: lexical search, Japanese search-text support, entity-aware retrieval, dense pgvector search, and RRF merge.
  • Benchmark: ICUBench-Hard-20 v2.
  • Gold reference source: GPT Pro, generated from the frozen Top-5 retrieved evidence.
  • Human-facing evaluation source: GPT Pro, using the same four-value rubric packet.
  • RAGAS scores: retained as reference diagnostics from an earlier pass; they were not rerun with GPT Pro gold answers.

The run tested seven model/mode cells: Qwen3.5 0.8B, 2B, and 4B non-thinking; Qwen3.5 0.8B and 4B thinking; Gemma 4 E2B non-thinking; and Gemma 4 E4B non-thinking.

Results

The active gold labels use three answerability levels.

  • Fully answerable: 10 questions.
  • Partially answerable: 8 questions.
  • Insufficient evidence: 2 questions.

The human-facing judge metrics are answer acceptance, readability, citation usefulness, and over-answering risk.

In this run, Qwen3.5 4B non-thinking has the best Qwen profile. Gemma 4 receives useful citation scores, but some outputs have readability and format problems. The thinking-mode saved final answers are not reliable enough to treat as final student-facing answers.

Reference automatic scores

The RAGAS plot is kept as a reference diagnostic from the earlier scoring pass. The current interpretation should prioritize the GPT Pro gold answers and GPT Pro human-facing judge scores.

The weak thinking-mode scores should be read as a harness and final-answer extraction issue, not as a general claim that thinking is worse.

Stability checks

These plots check variation across the 20 questions. They use a normalized human-facing composite score:

(answer acceptance + readability + citation usefulness + (2 - over-answering risk)) / 8

The score ranges from 0 to 1, where higher is better. The two thinking-mode runs are excluded because their saved final answers were not usable in this run.

The box plot shows question-by-question variation for each model.

The heatmap shows which questions create failures.

The confidence interval is computed with bootstrap resampling over the 20 questions.

Supporting checks

These figures are kept for inspection but should not replace the main interpretation.

Latency and length

Mean retrieval latency was 0.317s, with maximum retrieval latency 2.436s. Generation latency varied more by model and mode.

Thinking mode was slower in this run: Qwen3.5 0.8B thinking averaged about 21.878s, and Qwen3.5 4B thinking averaged about 30.259s.

Answer length matters because long answers can look useful while still receiving weak student-facing scores.

Interpretation

This run is usable as the final controlled baseline with one caveat: the GPT Pro references are research references, not official ICU answers.

The system worked end to end: local retrieval, local answer models, GPT Pro gold generation, GPT Pro human-facing judging, plots, and downloadable artifacts all completed.

The safest thesis claim is narrow. This is a fixed-context generation evaluation over a frozen retrieval cache. It tests whether each local model can synthesize a student-facing answer from the same evidence. It does not test independent retrieval quality for each model.

Next step

  • Use the GPT Pro gold and evaluation tables as the active final baseline.
  • Treat thinking-mode scores cautiously until final-answer extraction is consistent.
  • Select 5-8 representative failure cases for the thesis.
  • Use this run to explain why manual audit is needed alongside automatic metrics.
