Summary

This update reports the active final controlled evaluation data after replacing the older Qwen-35B gold/evaluation layer with GPT Pro generated gold answers and GPT Pro human-facing evaluation.

  • Run ID: final_controlled_v2_20260507T114145Z.
  • Runtime: local GPU evaluation environment with local PostgreSQL and pgvector.
  • Corpus: structural_fallback_v1, 30,216 passages.
  • Retrieval embedding: local BAAI/bge-m3.
  • Retrieval stack: lexical search, Japanese search-text support, entity-aware retrieval, dense pgvector search, and RRF merge.
  • Benchmark: ICUBench-Hard-20 v2.
  • Gold reference source: GPT Pro, generated from the frozen Top-5 retrieved evidence.
  • Human-facing evaluation source: GPT Pro, using the same four-value rubric packet.
  • RAGAS scores: retained as reference diagnostics from the earlier pass; they were not rerun with GPT Pro gold answers.
  • Tested generation models were run locally.

The active gold labels now use a three-way answerability scheme: fully answerable, partially answerable, and insufficient evidence.

Reference Automatic Scores

The RAGAS plot below is retained as a reference diagnostic from the previous scoring pass. It is useful for comparison, but the current final-round interpretation should prioritize the GPT Pro gold answers and GPT Pro human-facing judge scores below.

The Qwen thinking runs still have invalid or weak final-answer extraction behavior, so those scores should be treated as a harness issue rather than evidence that thinking is intrinsically worse.

Human-Facing Judge Scores

The human-facing judge metrics were answer acceptance, readability, citation usefulness, and over-answering risk. These are now based on the GPT Pro evaluation packet.

The clearest trend is that Qwen3.5 4B non-thinking has the strongest overall human-facing profile among the Qwen runs, while Gemma 4 outputs receive strong citation scores but weaker readability due to format/language issues. The thinking-mode runs fail as final student-facing answers in this setup because the saved final answers are incomplete or not usable.

Score Distribution and Stability

The following plots check stability across the 20 questions, not just model-level averages. They use a normalized human-facing composite score:

(answer acceptance + readability + citation usefulness + (2 - over-answering risk)) / 8

The score ranges from 0 to 1, where higher is better. The two thinking-mode runs are excluded here because their saved final answers were not usable in this run.

The box plot shows whether a model is stable across questions. A narrow box means the model behaves more consistently; a wide box means stronger question-by-question variation.

The heatmap shows which individual questions cause failures. This is useful because two models can have similar averages while failing on different question types.

The confidence interval is computed with bootstrap resampling over the 20 questions. It does not assume a normal distribution.

The next plots keep the same 5 May 7 non-thinking model cells and show the empirical distribution more directly. They do not use the May 11 agentic runtime data.

The bucket chart counts how many answers fall below 0.50, between 0.50 and 0.74, or at least 0.75 on the normalized composite score. This is more appropriate than fitting a normal curve to ordinal judge scores.

Here a student-facing pass means answer_acceptance = 2, readability >= 1, citation_usefulness >= 1, and over_answering_risk <= 1. The rolling window is 5 questions.

The failure-tag chart shows what kinds of errors each model makes, instead of only reporting a score.

The paired-delta plot compares every non-thinking model against Qwen3.5 4B question by question. This is useful because model averages can look close even when they fail on different questions.

Latency

Mean retrieval latency was 0.317s, with maximum retrieval latency 2.436s. Generation latency varied more by model and mode.

Thinking mode was much slower in this run: Qwen3.5 0.8B thinking averaged about 21.878s, and Qwen3.5 4B thinking averaged about 30.259s. This latency cost matters for the cost-aware system argument.

Answer length is useful here because long outputs can look superficially helpful while still receiving poor student-facing scores. This is one reason the thesis treats latency, verbosity, and human-facing review as separate layers.

Interpretation

This run is now usable as the final controlled baseline, with one caveat: the GPT Pro gold references are research references created from frozen retrieved evidence, not official ICU answers.

The system itself worked end to end: local retrieval, local answer models, GPT Pro gold generation, GPT Pro human-facing judging, plots, and downloadable artifacts all completed. The main finding is not that one metric cleanly selects a winner. The main finding is that automatic metrics, human-facing scores, latency, and output length reveal different failure modes.

For thesis writing, the safest claim is that this is a fixed-context generation evaluation over a frozen retrieval cache. It evaluates whether each local model can synthesize a student-facing answer from the same retrieved evidence, not whether each model performs retrieval independently.
