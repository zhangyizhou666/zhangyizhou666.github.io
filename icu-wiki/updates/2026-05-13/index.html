System30 Model and Mode Sweep

Remote pgvector/BGE runtime was evaluated on 30 regression questions across six small models and two answer modes.

evaluationsystem30agentic ragmodel sweep

Purpose

This update records a system-level regression run after moving the runtime to the remote ICU Wiki server. The goal was to test the current retrieval and answer path on questions that are closer to real student use than the earlier benchmark.

Evaluation setup

  • The benchmark contains 30 questions: 20 inherited hard cases and 10 new regression cases.
  • The new cases focus on English mandatory-course questions, ChatGPT and senior-thesis policy boundaries, teacher-name lookup in Japanese and Chinese, and course-registration completion procedures.
  • The runtime used PostgreSQL hybrid retrieval, local BGE-M3 embeddings, local BGE reranking, Janome tokenization, entity-aware retrieval, source-authority scoring, and a reranker candidate limit of 40.
  • The sweep tested Qwen3.5 0.8B, 2B, 4B, 9B, Gemma 4 E2B, and Gemma 4 E4B.
  • Each model was run in single-shot mode and agentic mode, giving 360 total answers.

Main result

  • Qwen3.5 9B was the strongest setting in this run.
  • Qwen3.5 9B agentic reached the best average human-quality score and the best clean-pass rate.
  • Qwen3.5 9B single-shot was close in quality and much faster on average.
  • Gemma 4 E2B and E4B were competitive on readability and citation presentation, but more often missed key conditions.
  • Qwen3.5 0.8B was below the current practical threshold for a student-facing answer path.
  • A separate model-selection screen now checks the smallest model that clears a quality floor while avoiding blocking bad cases.
  • The answer-length pattern is mixed. Qwen3.5 2B agentic has a shorter median answer than single-shot, but repeated or truncated bad cases make its mean answer length longer.
  • On the hardest questions, agentic mode is not uniformly helpful. Qwen3.5 9B is the clearest positive case, while Qwen3.5 2B and 4B add blocking bad cases on the hardest subset.

Interpretation

Agentic mode is useful when the question needs query planning, evidence judgment, or a second retrieval attempt. The cost is latency. For the current system, the clearest tradeoff is between Qwen3.5 9B single-shot as the faster strong baseline and Qwen3.5 9B agentic as the higher-quality diagnostic path.

The remaining weak cases are not only model-size problems. Several failures come from policy boundary questions where the retrieved text is partial, old, or too indirect. These cases need stricter abstention and better evidence selection, not just a larger model.

For model selection, average quality is not enough. A condition with a good average score can still be unusable if it has repeated, truncated, or acceptance-zero answers on specific questions. The new screen uses a working human-quality floor of 0.80 and separately counts blocking bad cases.

The new agentic-effect screen compares each model against itself, question by question. It treats the eight lowest-average-quality questions as the hardest subset. The result is diagnostic rather than statistical: each question has only six paired model comparisons, so the plots show where to investigate next, not a final proof that agentic mode helps or hurts.

Model selection

Smallest capable model screen

The decision target is not the largest score. It is the smallest model that clears a quality floor while avoiding bad cases that would block a student-facing release. Here, a blocking bad case means answer_acceptance = 0, human_quality < 0.50, or a human note that flags repetitive or truncated output.

Using a working quality floor of 0.80 and requiring zero blocking bad cases, the smallest passing condition is Gemma E4B single-shot. This is a screening result, not a final deployment claim.

Answer length

Agentic mode does not have one length pattern

The answer-length hypothesis is mixed in this run. Qwen3.5 2B agentic has a shorter median answer than single-shot, but a longer mean because repeated or truncated bad outputs pull the average upward. Qwen3.5 0.8B is roughly split rather than clearly shorter. Qwen3.5 9B and Gemma 4 E4B are longer in agentic mode on average.

Agentic effect

Where did agentic mode help?

This view compares each model against itself: agentic output minus single-shot output for the same model and question. The hardest screen uses the eight questions with the lowest average human quality across all model/mode outputs.

The signal is mixed. Qwen3.5 9B is the clearest positive case on the hardest questions: higher average quality and fewer blocking cases. Gemma 4 E2B and E4B improve on average, but the 4B Gemma still adds one blocking case over the full 30-question run. Qwen3.5 2B and 4B get worse on the hardest subset and add blocking cases. With only six paired model comparisons per question, this is enough for error analysis, but not enough for a statistical claim.
