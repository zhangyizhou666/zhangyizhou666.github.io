Benchmark history

Initial 50-question Gemma/Qwen evaluation

This was the first broad model comparison used for the seminar update. It is still useful as historical evidence, but the questions were often too simple to expose real chatbot failures.

Questions50

Broad ICU information questions, mostly direct and answerable. The set is easier than the later hard realistic benchmark.

ModelsGemma 3 + Qwen3.5

Gemma answers were generated through OpenRouter API calls; Qwen3.5 answers were generated locally.

UsefulnessHistorical baseline

The score pattern helped us discover that the original questions were too naive for the thesis claim.

Provenance

Question provenance

The saved questions.jsonl file does not preserve a provenance field, so we should treat these as a fixed benchmark rather than claim that the exact 50 questions were generated by RAGAS.

RAGAS did not generate questions during scoring. Scoring consumed fixed questions, retrieved contexts, and model answers.

The later questions_with_gold.jsonl and gold_answers_gpt5_mini.jsonl files did use a teacher model: openai/gpt-5-mini generated gold answers and answerability labels from fixed top-5 hybrid retrieved evidence.
